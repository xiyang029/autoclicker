# 项目结构

## 目标
将 `lib` 统一为 Feature-First 结构，降低跨目录跳转成本，并让业务归属清晰。

## 顶层约束
```text
lib/
├─ main.dart
└─ features/
```

## 当前约定

### `features/autoclicker`
- `controllers/`：页面状态编排、平台能力调度。
- `models/`：配置实体与数据转换。
- `pages/`：控制页、配置页、首页容器。
- `services/`：自动点击器专属平台通道与默认配置。
- `widgets/`：自动点击器复用组件。

### `features/setting`
- `pages/`：设置页与更新页展示。
- `services/`：版本检查、下载、安装权限相关服务。

## 落地规则
- 一个文件只放一个主职责。
- 目录命名优先使用职责名，避免 `utils`、`common`、`manager` 这类宽泛命名。
- 仅当两个及以上 feature 稳定复用后，才评估是否抽公共层。
- 页面只依赖本 feature 的 `controllers/services/widgets/models`，避免反向依赖其它 feature 的页面。

## 本次调整标准
- `lib/core` 下的更新能力迁移到 `features/setting/services/`。
- `lib/platform` 下的自动点击平台通道迁移到 `features/autoclicker/services/`。
- `lib` 顶层不再保留 `core/`、`platform/` 目录。
